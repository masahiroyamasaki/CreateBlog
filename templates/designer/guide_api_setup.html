概要・必要なもの

Instagram / Threads に自動投稿するには、Meta（Facebook）の開発者機能を使ってAPIアクセストークンを取得する必要があります。取得後、このアプリの企業設定画面に貼り付けるだけで自動投稿が有効になります。

必要なもの（事前に準備）
  • Facebookアカウント — 個人アカウントでOK
  • Instagramビジネスアカウント — 個人アカウントから変更可能
  • Facebookページ — Instagramビジネスアカウントと連携が必要
  • スマートフォン — Instagram/Threadsアプリが必要な手順あり
Instagramの個人アカウントではAPIは使えません。ビジネスアカウントまたはクリエイターアカウントへの変更が必須です。

Meta 開発者環境の構築

Meta Developer
1Instagramをビジネスアカウントに変更
Instagramアプリ → プロフィール → 右上メニュー → 「アカウントの種類とツール」 → 「プロアカウントに切り替える」→ ビジネスを選択
クリエイターアカウントでも可ですが、ビジネスアカウントの方が安定しています。
2FacebookページをInstagramと連携
  1. Facebookで新しいページを作成（または既存ページを使用）
  2. Instagramアプリ → プロフィール → 「プロフィールを編集」
  3. 「ソーシャルリンクを追加」→「Facebookページをリンク」で連携
この連携がないとInstagram Graph APIが動作しません。必ず先に行ってください。
3Meta for Developers でアプリを作成
  1. developers.facebook.com にアクセスし、Facebookアカウントでログイン
  2. 「マイアプリ」→「アプリを作成」をクリック
  3. ユースケース：「その他」を選択 → 次へ
  4. アプリタイプ：「ビジネス」を選択 → 次へ
  5. アプリ名を入力（例：MyBlog Poster）→ 「アプリを作成」
4Instagram Graph API 製品を追加
  1. 作成したアプリのダッシュボードで「製品を追加」をクリック
  2. 「Instagram Graph API」→「設定」をクリック
  3. 左メニューに「Instagram」が追加される

Instagram API の設定

Instagram
5Instagram ビジネスアカウントIDとトークンを取得
  1. アプリダッシュボード → 左メニュー「Instagram」→「APIのセットアップ（Instagram Graph API）」
  2. 「Instagramビジネスアカウントを追加または削除」→ 自分のInstagramアカウントを連携
  3. 「アクセストークンを生成」→ 表示されたトークンをコピー
  4. 「Instagramビジネスアカウントに接続されているページ」欄のIG User ID（数字）もコピー
コピーすべき2つの値
  • IG User ID（例：17841400000000000）→ アプリの「IGビジネスアカウントID」に入力
  • アクセストークン（EAA...で始まる長い文字列）→ アプリの「長期アクセストークン」に入力
6アクセストークンを長期トークンに変換（重要）

デフォルトのトークンは有効期限が約60日です。このアプリは長期トークンをそのまま保存します。定期的な更新が必要です。

トークンの有効期限について
約60日で失効します。失効するとInstagramへの自動投稿が停止します。有効期限が近づいたら企業設定から新しいトークンを発行・更新してください。

Meta for Developers の「アクセストークンデバッガー」（developers.facebook.com/tools/debug/accesstoken）でトークンの残り有効期間を確認できます。

Threads API の設定

Threads

ThreadsのAPIはInstagramとは別のフローです。Instagramアカウントと同じMetaアカウントで取得します。

7Threads API 製品を追加
  1. Metaアプリダッシュボード → 「製品を追加」
  2. 「Threads API」→「設定」をクリック
  3. 左メニューに「Threads」が追加される
8Threads アクセストークンを取得
  1. 左メニュー「Threads」→「クイックスタート」
  2. 「Threadsアカウントを追加」→ Threadsにログインして連携を許可
  3. 「アクセストークンを生成」→ トークンをコピー
コピーすべき値
  • Threads User ID（数字）→ 企業設定の「Threads ユーザーID」に入力
    ※ 企業設定の「IDを自動取得」ボタンでトークンから自動取得することも可能
  • アクセストークン → 企業設定の「Threadsアクセストークン」に入力
Threads投稿には threads_content_publish 権限が必要です。アプリが「開発モード」の場合はテストユーザーのみ投稿可能です。本番公開が必要な場合はMetaの審査を申請してください。

アプリへの設定方法

9企業設定画面に入力
企業一覧 → 該当企業の「設定編集」を開き、以下に入力して保存します。
  • 投稿タイプ：Instagram を選択
  • IGビジネスアカウントID — IG User ID（手順5でコピーした数字）
  • 長期アクセストークン — Instagram APIトークン（手順5でコピー）
  • Threadsユーザーid — Threads User ID（手順8）または「IDを自動取得」
  • Threadsアクセストークン — Threads APIトークン（手順8でコピー）

よくあるトラブル

「投稿に失敗しました」と表示される
  • トークンが失効している可能性が高いです。Meta Developer でトークンを再発行し、企業設定を更新してください。
  • Instagramアカウントがビジネスアカウントに変更されていない場合も発生します。
  • FacebookページとInstagramの連携が解除されていないか確認してください。
「IG User IDが見つからない」
  • Instagram がビジネスアカウントになっていることを確認
  • FacebookページとInstagramが正しく連携されていることを確認
  • Metaアプリのダッシュボードで、Instagramアカウントが「連携済み」になっているか確認
Threadsの「threads_content_publish 権限がありません」
  • アプリが開発モードの場合、投稿できるのはアプリに「テストユーザー」として追加されたアカウントのみです
  • 自分のアカウントをテストユーザーに追加：アプリダッシュボード →「役割」→「テストユーザー」→「テストユーザーを追加」
  • それでも解決しない場合はMetaに権限の審査を申請する必要があります
